Can be added to any GameObject with an interactable binary state in its parents.

If players should not be allowed to interact with the object in the ordinary manner, add the Interactability_Remote flag to its asset to indicate only mod hooks should control it.

Should the OnStateEnabled and OnStateDisabled events be invoked when the object is loaded, becomes relevant in multiplayer, and is reset? True is useful when visuals need to be kept in sync with the state, whereas false is useful for transient interactions.

ListenServerHostMode

Controls how state change requests are performed when running as both client and server ("listen server"). On the dedicated server, requesting a state change overrides the current state without processing NPC conditions, whereas when a client requests a state change NPC conditions apply. This option fixes the inconsistency in singleplayer of whether to treat as server or client. (public issue #4298) At the time of writing (2024-01-29) listen server only applies to singleplayer.
