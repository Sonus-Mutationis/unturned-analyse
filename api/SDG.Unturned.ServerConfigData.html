Fake_Lag_Threshold_Seconds

Ordinarily the server should be receiving multiple input packets per second from a client. If more than this amount of time passes between input packets we flag the client as potentially using a lag switch, and modify their stats (e.g. reduce player damage) for a corresponding duration. Minimum value is PlayerInput.MIN_FAKE_LAG_THRESHOLD_SECONDS.
